Руководство по функциям библиотеки SORT(3)

НАЗВАНИЕ
        sort, sort_set_method, sort_set_direction - универсальная библиотека для сортировки массивов на месте

КРАТКОЕ ОПИСАНИЕ
        #include sort.h

        int sort(void *array, int size, int elemsize,
        int (*comp)(void *a1, void *a2));

        int sort_set_method(sort_method_t method);

        int sort_set_direction(sort_direction_t direction);

        Компиляция и компоновка с помощью:

        gcc -o program program.c sort.c

ОПИСАНИЕ
        sort() сортирует массив, на который указывает array, на месте, используя алгоритм
        и направление, ранее выбранные функциями sort_set_method() и sort_set_direction().

        Массив содержит size элементов, каждый размером elemsize байт. Библиотека
        использует memcpy(3) внутри себя, поэтому она корректно работает для любого типа элементов,
        включая структуры.

        Предоставленная пользователем функция comp вызывается для сравнения двух элементов и
        должна возвращать:

        -1, если элемент в a1 меньше элемента в a2

        0, если они равны

        1, если элемент в a1 больше элемента в a2

        sort_set_method() выбирает алгоритм сортировки. Этот параметр хранится
        в статической переменной внутри sort.c, невидимой для вызывающей стороны. Он сохраняется
        до следующего вызова sort_set_method().

        sort_set_direction() выбирает порядок сортировки. Когда SORT_DESC активен,
        библиотека автоматически инвертирует результат comp перед каждым сравнением,
        поэтому все алгоритмы выдают убывающий результат без каких-либо
        изменений в своем коде.

АЛГОРИТМЫ
        ПУЗЫРЬКОВАЯ СОРТИРОВКА
        Простая сортировка обменом. Многократно проходит по массиву и меняет местами соседние элементы, расположенные не по порядку. Временная сложность O(n^2) в среднем и в худшем случае.

        Сортировка вставками (INSERTION_SORT)
        Строит отсортированную последовательность по одному элементу за раз, вставляя каждый новый элемент в правильную позицию. Временная сложность
        O(n^2). Стабильная.

        Сортировка выбором (SELECTION_SORT)
        На каждом проходе находит минимальный элемент в несортированном хвосте и
        перемещает его в начало. Временная сложность O(n^2). Нестабильная.

        Сортировка комбинированными блоками (COMB_SORT)
        Улучшение по сравнению с пузырьковой сортировкой, использующее уменьшающийся зазор (в 1,3 раза) для удаления элементов, расположенных не по порядку и находящихся далеко друг от друга («черепашек»).
        На ранних этапах. Временная сложность O(n^2) в худшем случае, O(n log n) в среднем.

        Сортировка ободком (SHELL_SORT)
        Обобщение сортировки вставками. Сортирует элементы, разделенные зазором, который уменьшается вдвое на каждом проходе (n/2, n/4, ..., 1). Временная сложность -
        O(n^(3/2)) с этой последовательностью пробелов.

        БЫСТРАЯ СОРТИРОВКА
        Алгоритм «разделяй и властвуй». Разделяет массив вокруг
        последнего элемента в качестве опорного и рекурсивно обрабатывает обе половины. Временная сложность -
        O(n log n) в среднем, O(n^2) в худшем случае. Нестабильный.

        СОРТИРОВКА СЛИЯНИЯ
        Рекурсивно делит массив пополам, сортирует каждую половину, затем
        объединяет две отсортированные половины. Выделяет O(n) дополнительной памяти.

        Временная сложность O(n log n). Стабильный.

        СОРТИРОВКА КУЧИ
        Строит максимальную кучу из массива, затем многократно извлекает
        максимальный элемент. Временная сложность O(n log n). Выполняется на месте. Нестабильный.

ВОЗВРАЩАЕМОЕ ЗНАЧЕНИЕ
        Все три функции возвращают SORT_OK (0) в случае успеха и ненулевой код ошибки
        в случае неудачи.

        КОДЫ ОШИБОК
        SORT_OK (0)
        Успех.

        SORT_ERR_NULL (1)
        массив или компонент равен NULL.

        SORT_ERR_SIZE (2)
        размер или размер элемента <= 0.

        SORT_ERR_METHOD (3)
        Неизвестное значение sort_method_t передано в sort_set_method().

        SORT_ERR_DIRECTION (4)
        Неизвестное значение sort_direction_t передано в sort_set_direction().

        SORT_ERR_ALLOC (5)
        Внутренняя ошибка malloc(3).

ТИПЫ И КОНСТАНТЫ
        sort_method_t
        typedef enum {
        BUBBLE_SORT = 1,
        INSERTION_SORT = 2,
        SELECTION_SORT = 3,
        COMB_SORT = 4,
        SHELL_SORT = 5,
        QUICK_SORT = 6,
        MERGE_SORT = 7,
        HEAP_SORT = 8

        } sort_method_t;

        sort_direction_t
        typedef enum {
        SORT_ASC = 1,
        SORT_DESC = 2

        } sort_direction_t;

        Метод по умолчанию — BUBBLE_SORT, направление по умолчанию —
        SORT_ASC.

ПРИМЕРЫ
        Сортировка массива целых чисел

        #include <stdio.h>

        #include "sort.h"

        int comp_int(void *a, void *b)

        {
        int n1 = *((int *)a);
        int n2 = *((int *)b);
        if (n1 < n2) return -1;
        if (n1 > n2) return 1;
        return 0;

        }

        int main(void)

        {
        int arr[] = {5, 3, 1, 4, 2};
        int n = 5;

        sort_set_method(QUICK_SORT);

        sort_set_direction(SORT_ASC);

        if (sort(arr, n, sizeof(int), comp_int) != SORT_OK) {
        return 1;

        }

        /* Теперь arr: 1 2 3 4 5 */
        return 0;

        }

        Сортировка массива структур по полю типа double

        #include "sort.h"

        typedef struct {
        int id;
        double value;

        } item_t;

        int comp_item(void *a, void *b)

        {
        item_t *s1 = (item_t *)a;
        item_t *s2 = (item_t *)b;
        if (s1->value < s2->value) return -1;
        if (s1->value > s2->value) return 1;
        return 0;

        }

        int main(void)

        item_t items[] = {{3, 3.0}, {1, 1.0}, {2, 2.0}};

        sort_set_method(MERGE_SORT);
        sort_set_direction(SORT_DESC);
        sort(items, 3, sizeof(item_t), comp_item);

        /* результат: {3,3.0}, {2,2.0}, {1,1.0} */
        return 0;

        }

        /* Возвращает 1, если *a > *b, 2 в противном случае. */
        int max_elem(void *a, void *b, int (*compare)(void *, void *))

        {
        if (compare(a, b) == 1) return 1;

        return 2;

        }

ПРИМЕЧАНИЯ
        +o Текущий метод и направление хранятся в статических переменных внутри
        sort.c. Они невидимы за пределами этой единицы трансляции. Единственный способ изменить их — это использовать функции sort_set_method() и sort_set_direction(), которые проверяют входные данные перед их сохранением. Использование недопустимого значения перечисления приводит к тому, что эти функции возвращают ошибку и оставляют текущую настройку без изменений.

        +o Арифметика указателей на void * не определена в стандартном C. Библиотека
        всегда приводит к типу char * (один байт на единицу) перед вычислением смещений элементов.

        +o Для обмена элементами используется 256-байтовый буфер стека для небольших элементов, чтобы избежать вызова malloc(3) при каждом обмене. Выделение памяти в куче используется только тогда, когда размер элемента превышает 256 байт.

        +o Библиотека не является потокобезопасной. Если два потока одновременно вызывают sort_set_method() или sort(), поведение не определено.
